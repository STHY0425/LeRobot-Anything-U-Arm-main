# 🎯 舵机角度读取原理 - 简明版

## 核心原理（一句话）

**舵机内部有位置传感器，通过串口读取传感器的数值，然后转换成角度。**

---

## 📡 工作流程（3步）

```
1. 发送命令 → 2. 读取数值 → 3. 转换角度
   (串口)        (PWM/位置)      (度数)
```

---

## 🔧 两种舵机方案

### 方案 1: 中菱舵机（文本协议）

```python
# 发送命令
serial.write('#001PRAD!')  # 读取 1 号舵机位置

# 接收响应
response = 'P1500'  # PWM 值

# 转换角度
angle = (1500 - 500) / 2000 * 270  # = 135°
```

**原理**: PWM 值 500-2500 对应 0-270°

---

### 方案 2: 飞特舵机（二进制协议）

```python
# 读取位置
position = read_servo(id=1)  # 例如: 2047

# 转换角度
angle = (2047 - 2047) / 4096 * 360  # = 0°
```

**原理**: 12位编码器，4096个位置对应 360°

## 🔑 关键代码

### 读取角度

```python
# 1. 打开串口
ser = serial.Serial('/dev/ttyUSB0', 115200)

# 2. 读取舵机
def read_servo_angle(servo_id):
    # 发送命令
    ser.write(f'#{servo_id:03d}PRAD!'.encode())
    
    # 接收响应
    response = ser.read_all().decode()
    
    # 解析 PWM
    pwm = int(re.search(r'P(\d{4})', response).group(1))
    
    # 转换角度
    angle = (pwm - 500) / 2000 * 270
    
    return angle

# 3. 使用
angle = read_servo_angle(1)  # 读取 1 号舵机
print(f"角度: {angle}°")
```

### 零位校准

```python
# 启动时记录初始位置
zero_angle = read_servo_angle(1)  # 例如: 135°

# 运行时计算偏移
current_angle = read_servo_angle(1)  # 例如: 180°
offset = current_angle - zero_angle  # = 45°

# 发布偏移量
publish(offset)  # 发布 45°
```

## ❓ 常见问题

### Q1: 为什么需要零位校准？
**A**: 每次启动时舵机位置不同，需要记录初始位置作为参考点。

### Q2: 角度为什么会抖动？
**A**: 传感器读数有噪声，使用插值平滑可以改善。

### Q3: 如何提高精度？
**A**: 
- 使用更高分辨率的舵机（飞特 > 中菱）
- 增加波特率
- 使用滤波算法

### Q4: 支持多少个舵机？
**A**: 理论上无限，实际受限于：
- 串口通信速度
- 控制板地址数量
- 更新频率要求
