## ⚙️ 物理引擎工作原理

### 每个时间步（1/240秒）做什么？

```
┌─────────────────────────────────────────────────────┐
│  Step 1: 收集外力                                     │
│  • 重力: F_gravity = m * g                           │
│  • 控制力矩: τ_control (来自控制器)                   │
│  • 接触力: F_contact (碰撞产生)                       │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│  Step 2: 碰撞检测                                     │
│  • 宽相位: AABB包围盒快速排除                          │
│  • 窄相位: GJK算法精确检测                             │
│  • 计算接触点和法线                                    │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│  Step 3: 约束求解                                     │
│  • 关节约束 (旋转、平移限制)                           │
│  • 接触约束 (防止穿透)                                 │
│  • 摩擦约束 (库仑摩擦)                                 │
│  • 迭代求解约束力 λ                                   │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│  Step 4: 积分更新                                     │
│  • 速度更新: v_new = v_old + (F/m) * dt              │
│  • 位置更新: p_new = p_old + v_new * dt              │
│  • 角速度更新: ω_new = ω_old + (τ/I) * dt            │
│  • 姿态更新: q_new = q_old + ω_new * dt              │
└─────────────────────────────────────────────────────┘
```

## 🎮 PD控制器详解

### 什么是PD控制？

```
目标位置 (target)
    ↓
    ├─→ [比例项 P] ─→ Kp * (target - current) ─┐
    │                                          │
当前位置 (current)                              ├─→ [求和] → 力矩 τ
    │                                          │
    └─→ [微分项 D] ─→ -Kd * velocity ─────────┘
```

### 参数调优

```
┌─────────────────────────────────────────────────────┐
│  Kp (刚度) - 比例增益                                 │
│                                                      │
│  太小 ← ─────────── 合适 ─────────── → 太大          │
│  响应慢              稳定快速              震荡        │
│  误差大              误差小                不稳定      │
│                                                      │
│  推荐: 1e2 ~ 1e4                                     │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│  Kd (阻尼) - 微分增益                                 │
│                                                      │
│  太小 ← ─────────── 合适 ─────────── → 太大          │
│  震荡                稳定                响应慢        │
│  超调                平滑                过阻尼        │
│                                                      │
│  推荐: 1e1 ~ 1e3                                     │
└─────────────────────────────────────────────────────┘
```

### 代码示例

```python
# 简化的PD控制器
def pd_control(target, current, velocity, kp=1000, kd=100):
    """
    target: 目标关节角度
    current: 当前关节角度
    velocity: 当前关节速度
    """
    # 位置误差
    error = target - current
    
    # PD控制公式
    torque = kp * error - kd * velocity
    
    # 力矩限制（防止过大）
    torque = np.clip(torque, -100, 100)
    
    return torque

# 使用示例
target_angle = 1.57  # 90度
current_angle = 0.0
current_velocity = 0.0

torque = pd_control(target_angle, current_angle, current_velocity)
# torque = 1000 * 1.57 - 100 * 0 = 1570 N·m
```

## 📊 三个示例程序对比

### 1. static_robot_viewer.py - 静态查看

```
功能: 显示机器人模型
特点: 最简单，只渲染不控制

流程:
创建环境 → 设置姿态 → 循环渲染 → 关闭

代码:
env = gym.make("Empty-v1", robot_uids="panda")
while True:
    env.render()  # 只渲染，不执行动作
```

### 2. sin_xarm.py - 简单运动

```
功能: 机器人在两个姿态间往复运动
特点: 预定义轨迹，线性插值

流程:
创建环境 → 定义姿态A和B → 循环插值 → 执行动作

代码:
pose_a = [0, 0, 0, 0, 0, 0, 0]
pose_b = [1, 1, 1, 1, 1, 1, 0]
for t in linspace(0, 1, 200):
    action = (1-t)*pose_a + t*pose_b
    env.step(action)
```

### 3. teleop_sim.py - 遥操作

```
功能: 实时读取硬件舵机，控制仿真机器人
特点: 双线程，实时同步

流程:
创建环境 → 启动读取线程 → 启动控制线程 → 实时同步

架构:
[舵机硬件] → [串口读取] → [队列] → [角度映射] → [仿真控制]
   50Hz         线程1      线程安全    转换逻辑      线程2
```

## 🔍 关键概念速查

### URDF (Unified Robot Description Format)
```xml
机器人描述文件，包含:
• 连杆 (link): 刚体部件
• 关节 (joint): 连接方式
• 网格 (mesh): 3D模型
• 物理属性: 质量、惯性
```

### 关节类型
```
• revolute: 旋转关节 (有限位)
• continuous: 连续旋转 (无限位)
• prismatic: 平移关节
• fixed: 固定连接
```

### 控制模式
```
• pd_joint_pos: 关节位置控制
• pd_joint_delta_pos: 关节增量控制
• pd_ee_pose: 末端位姿控制
• pd_joint_vel: 关节速度控制
```

### 观察模式
```
• state: 状态向量 (关节角度等)
• rgb: RGB图像
• depth: 深度图
• pointcloud: 点云
• sensor_data: 原始传感器数据
```

## 💡 快速理解要点

### 1. 仿真 = 物理 + 渲染

```
物理引擎 (PhysX)          渲染引擎 (Vulkan)
     ↓                          ↓
计算运动和碰撞              生成视觉图像
     ↓                          ↓
更新位置/速度               显示在屏幕上
```

### 2. 控制 = 目标 → 力矩 → 运动

```
用户输入目标 → PD控制器计算力矩 → 物理引擎模拟运动
```

### 3. 时间步 = 1/240秒

```
每秒执行240次物理计算
每次计算推进时间 4.17ms
```

### 4. 线程模型（遥操作）

```
主线程: 管理和协调
   ↓
生产者线程: 读取硬件 (50Hz)
   ↓
队列: 线程安全通信
   ↓
消费者线程: 控制仿真 (50Hz)
```
