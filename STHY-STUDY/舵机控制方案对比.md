# 🎮 舵机控制仿真方案对比

## 📊 三种方案对比

| 方案 | 需要硬件 | 实时性 | 难度 | 适用场景 |
|------|---------|--------|------|---------|
| **真实舵机控制** | ✅ 需要 U-Arm | ⭐⭐⭐⭐⭐ 最高 | ⭐⭐⭐ | 实际应用、数据采集 |
| **模拟舵机演示** | ❌ 不需要 | ⭐⭐⭐ 中等 | ⭐ | 学习、测试、演示 |

## 1️⃣ 真实舵机控制 - `teleop_sim.py`

### 特点
- ✅ 真实硬件输入
- ✅ 实时同步（< 20ms 延迟）
- ✅ 双线程架构
- ✅ 支持多种机器人

### 硬件要求
```
U-Arm 主操作臂
  ├─ 7个舵机（中林/飞特）
  ├─ USB转串口线
  └─ 12V电源
```

### 使用方法
```bash
# 连接硬件
sudo chmod 666 /dev/ttyUSB0

# 运行
cd src/simulation
python3 teleop_sim.py --robot panda --serial-port /dev/ttyUSB0
```

### 工作流程
```
物理舵机 → 串口读取 → 角度转换 → 队列传递 → 仿真控制
  (50Hz)     (线程1)     (PWM→rad)   (线程安全)   (线程2)
```

### 优点
- 真实的触觉反馈
- 自然的操作体验
- 可用于数据采集
- 支持力反馈（扩展）

### 缺点
- 需要购买硬件（~$60起）
- 需要组装和调试
- 串口配置可能有问题

## 2️⃣ 模拟舵机演示 - `teleop_sim_demo.py` ⭐ 推荐新手

### 特点
- ✅ 无需硬件
- ✅ 多种运动模式
- ✅ 即开即用
- ✅ 适合学习

### 运动模式

#### 正弦波模式（sine）
```bash
python3 teleop_sim_demo.py --robot panda --mode sine
```
效果：关节做正弦波运动，平滑往复

#### 圆周模式（circle）
```bash
python3 teleop_sim_demo.py --robot panda --mode circle
```
效果：末端画圆，展示协调运动

#### 波浪模式（wave）
```bash
python3 teleop_sim_demo.py --robot panda --mode wave
```
效果：关节依次运动，像波浪传递

#### 随机模式（random）
```bash
python3 teleop_sim_demo.py --robot panda --mode random
```
效果：随机探索，测试工作空间

### 优点
- 零成本，立即可用
- 多种预设运动
- 适合演示和学习
- 代码简单易懂

### 缺点
- 非交互式
- 运动轨迹固定
- 无法自由控制

## 3️⃣ 键盘交互控制 - `teleop_sim_demo.py --mode interactive`

### 特点
- ✅ 无需硬件
- ✅ 手动控制
- ✅ 精确调整
- ✅ 适合调试

### 使用方法
```bash
python3 teleop_sim_demo.py --robot panda --mode interactive
```

### 键盘操作
```
1-7/8  : 选择要控制的关节
w/s    : 增加/减少角度（小步长）
a/d    : 快速调整（大步长）
r      : 重置到 home 姿态
h      : 显示帮助
q      : 退出
```

### 使用场景
```
调试关节映射 → 测试关节限位 → 验证运动学 → 记录关键帧
```

### 优点
- 精确控制每个关节
- 适合调试和测试
- 可以记录关键姿态
- 无需额外硬件

### 缺点
- 操作不够直观
- 需要逐个关节调整
- 无法同时控制多关节

## 💡 常见问题

### Q1: 我没有硬件，能用吗？

**A**: 约 20ms（50Hz 控制频率），人眼感觉不到延迟。
