# ROS容器永久依赖安装方案

## 🎯 三种方案对比

### 方案1：自定义Docker镜像（最推荐⭐⭐⭐⭐⭐）
**优点：**
- 依赖永久保存在镜像中
- 启动速度最快（秒开）
- 可以分享给其他人使用
- 专业、标准的做法

**缺点：**
- 首次构建需要10-20分钟
- 占用额外磁盘空间（约6-7GB）

**使用步骤：**
```bash
# 1. 构建自定义镜像（只需做一次）
~/build_custom_image.sh

# 2. 以后每次启动
~/run_ros_custom.sh
```

---

### 方案2：保留容器不删除（简单⭐⭐⭐⭐）
**优点：**
- 无需构建镜像
- 立即可用

**缺点：**
- 容器损坏需要重新安装
- 不能分享给别人

**使用步骤：**
```bash
# 使用修改后的脚本（已去掉--rm）
~/run_ros_tmux_v2.sh
```

---

### 方案3：挂载本地Python环境（不推荐⭐⭐）
**优点：**
- 可以在宿主机和容器间共享包

**缺点：**
- 容易出现版本冲突
- 不够隔离
- 配置复杂

---

## 📋 推荐流程

### 如果你想要最佳体验：
```bash
# 1. 构建自定义镜像
~/build_custom_image.sh

# 等待10-20分钟...

# 2. 启动容器
~/run_ros_custom.sh

# 以后每次启动都是秒开！
```

### 如果你想快速开始：
```bash
# 直接使用保留容器的方案
~/run_ros_tmux_v2.sh
```

## 🔧 管理命令

### 查看所有镜像
```bash
sudo docker images
```

### 查看所有容器
```bash
sudo docker ps -a
```

### 删除旧容器（重新开始）
```bash
sudo docker stop ros_noetic
sudo docker rm ros_noetic
```

### 删除自定义镜像（释放空间）
```bash
sudo docker rmi ros_noetic_custom:latest
```

## 💡 常见问题

**Q: 构建镜像需要多久？**
A: 10-20分钟，取决于网速。但只需要做一次。

**Q: 镜像占用多少空间？**
A: 约6-7GB（包含ROS + 所有依赖）

**Q: 可以修改依赖吗？**
A: 可以！编辑 ~/Dockerfile.ros_custom，然后重新构建。

**Q: 两个方案可以共存吗？**
A: 可以！它们使用不同的容器名称。
