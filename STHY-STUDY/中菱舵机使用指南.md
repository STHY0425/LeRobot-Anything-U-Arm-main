# 🔧 中菱舵机使用指南（精简版）

> **核心流程**: 配置ID → 验证 → 仿真测试

---

## ⚠️ 重要说明

**中菱舵机出厂时所有 ID 都是 `000`，必须先配置 ID 才能使用！**

```
问题：所有舵机 ID 都是 000
电脑发送: #000PRAD!
    ↓
舵机1、2、3...7 全部同时响应 ❌

解决：配置不同的 ID (0-6)
电脑发送: #001PRAD!
    ↓
只有舵机1响应 ✅
```

**ID 与安装位置对应关系（从下到上）：**
```
ID 6 ← 夹爪（最顶部）
ID 5 ← 腕关节3
ID 4 ← 腕关节2
ID 3 ← 腕关节1
ID 2 ← 肘关节
ID 1 ← 肩关节
ID 0 ← 基座（最底部）
```

## 📋 准备工作

### 1. 环境准备

```bash
# 检查 Python 环境
python3 --version  # 需要 3.6+

# 进入项目目录
cd ~/2026robotic/LeRobot-Anything-U-Arm-main

# 安装依赖
pip3 install pyserial

# 安装 ROS 依赖（用于仿真测试）
pip3 install -r ros1_requirements.txt
```

### 2. 硬件连接

```bash
# 检查串口设备
ls -l /dev/ttyUSB*
# 输出: crw-rw---- 1 root dialout 188, 0 Jan 22 10:30 /dev/ttyUSB0

# 添加串口权限（二选一）
# 方法A：临时权限（重启后失效）
sudo chmod 666 /dev/ttyUSB0

# 方法B：永久权限（推荐）
sudo usermod -a -G dialout $USER
# 然后重新登录
```

**硬件连接示意：**
```
电源适配器(7.4V/12V) → 舵机控制板(VCC/GND) → 舵机(信号/VCC/GND)
电脑 → USB转TTL线(TX/RX/GND) → 舵机控制板(RX/TX/GND)

⚠️ 注意：TX接RX，RX接TX（交叉连接）
```

## 第一步：配置舵机 ID

### ⚠️ 重要规则

**每次只连接一个舵机！**
- 出厂所有舵机 ID 都是 000
- 如果同时连接多个，它们会同时响应命令
- 必须逐个配置：配置一个 → 断开 → 连接下一个

### 操作步骤

#### 1. 进入脚本目录

```bash
cd ~/2026robotic/LeRobot-Anything-U-Arm-main/src/uarm/scripts/Uarm_teleop/Zhonglin_servo

# 查看可用脚本
ls -la
# 输出:
# setup_servos.py    ← 自动化配置工具（我们要用这个）
# servo_changeid.py  ← 手动配置脚本
# servo_zero.py      ← 测试脚本
# servo_reader.py    ← ROS节点
```

#### 2. 运行配置工具

```bash
python3 setup_servos.py
```

**输出：**
```
======================================================================
                     中菱舵机配置向导
======================================================================

✅ 串口 /dev/ttyUSB0 可用

----------------------------------------------------------------------
请选择操作:
  1. 配置舵机 ID (首次使用必须)
  2. 测试所有舵机
  3. 释放所有舵机扭矩 (便于手动调整)
  4. 锁定所有舵机扭矩
  5. 退出
----------------------------------------------------------------------

请输入选项 (1-5):
```

#### 3. 输入 1 开始配置

```bash
1  # 输入 1 并按 Enter
```

**输出：**
```
======================================================================
【配置舵机 ID】
======================================================================

⚠️  重要说明:
  1. 每次只连接一个舵机到控制板
  2. 配置完成后断开该舵机
  3. 再连接下一个舵机
  4. 按顺序配置 ID: 0, 1, 2, 3, 4, 5, 6

为什么要这样做？
  - 出厂舵机 ID 都是 000
  - 如果同时连接多个，它们会同时响应命令
  - 必须逐个配置不同的 ID

按 Enter 开始配置...
```

#### 4. 配置第 1 个舵机（ID 0）

**按 Enter 继续，输出：**
```
======================================================================
配置舵机 1/7 (ID = 0)
======================================================================

1. 请只连接第 1 个舵机
2. 确认连接后按 Enter 开始配置...
```

**【硬件操作】**
1. 确保只有一个舵机连接到控制板
2. 检查舵机电源线、信号线都已连接
3. 按 Enter

**输出：**
```
正在配置 ID 0...
✅ 舵机 ID 0 配置完成
✅ 验证成功: 当前角度 = 135.0°

3. 请断开第 1 个舵机，按 Enter 继续...
```

**【硬件操作】**
1. 拔掉第 1 个舵机的信号线
2. 按 Enter

#### 5. 配置第 2 个舵机（ID 1）

**输出：**
```
======================================================================
配置舵机 2/7 (ID = 1)
======================================================================

1. 请只连接第 2 个舵机
2. 确认连接后按 Enter 开始配置...
```

**【硬件操作】**
1. 连接第 2 个舵机
2. 按 Enter

**输出：**
```
正在配置 ID 1...
✅ 舵机 ID 1 配置完成
✅ 验证成功: 当前角度 = 120.5°

3. 请断开第 2 个舵机，按 Enter 继续...
```

#### 6. 重复配置剩余舵机

按照相同步骤配置 ID 2, 3, 4, 5, 6：
- 断开上一个舵机
- 连接新舵机
- 按 Enter 配置
- 看到 ✅ 表示成功

**最后输出：**
```
======================================================================
🎉 所有舵机 ID 配置完成！
======================================================================

下一步:
  1. 连接所有 7 个舵机
  2. 选择菜单选项 2 测试所有舵机
```

## 第三步：仿真测试

配置验证成功后，测试舵机是否能控制仿真机械臂同步运动。

### 1. 启动 ROS 核心

**打开终端 1：**
```bash
roscore
```

**输出：**
```
... logging to /home/user/.ros/log/xxx/roscore-xxx.log
...
started core service [/rosout]
```

⚠️ **保持这个终端运行，不要关闭！**

### 2. 启动舵机读取节点

**打开终端 2：**
```bash
cd ~/2026robotic/LeRobot-Anything-U-Arm-main

# 启动舵机读取节点
rosrun uarm servo_reader.py
```

**输出：**
```
[INFO] [1737561234.567890]: Serial port opened
[INFO] [1737561234.678901]: Servo initial angle calibration completed
```

⚠️ **保持这个终端运行！**

### 3. 验证 ROS 话题

**打开终端 3：**
```bash
# 查看所有话题
rostopic list
```

**输出：**
```
/rosout
/rosout_agg
/servo_angles  ← 应该看到这个话题
```

**查看话题数据：**
```bash
rostopic echo /servo_angles
```

**输出：**
```
data: [0.0, -0.5, 1.2, 0.3, -0.8, 0.0, 0.0]
---
data: [0.1, -0.5, 1.3, 0.3, -0.7, 0.0, 0.0]
---
...（持续输出）
```

**验证：**
- 移动 U-Arm 舵机，应该能看到数值实时变化 ✅
- 按 `Ctrl+C` 停止查看

## 🔧 故障排除

### 问题 1: 找不到串口设备

```bash
# 错误: ls: cannot access '/dev/ttyUSB0': No such file or directory

# 解决方法 1: 查找正确的设备名
ls -l /dev/tty*
# 可能是 /dev/ttyUSB1 或 /dev/ttyACM0

# 解决方法 2: 检查 USB 连接
dmesg | grep tty | tail -10
# 查看最近连接的串口设备

# 解决方法 3: 检查 USB 线
# - 尝试更换 USB 端口
# - 尝试更换 USB 线
```

### 问题 2: 舵机无响应

```bash
# 运行 servo_zero.py 没有输出或全是 None

# 检查 1: 电源
# - 确保电源适配器已连接
# - 检查电压（7.4V 或 12V）
# - 测量舵机控制板的电源指示灯是否亮

# 检查 2: 串口连接
# - TX 接 RX，RX 接 TX（交叉连接）
# - GND 必须连接
# - 检查接线是否松动

# 检查 3: 波特率
# 确认波特率为 115200
```

### 问题 3: 部分舵机无响应

```bash
# 测试显示 X/7 舵机响应正常 (X < 7)

# 解决方法: 重新配置未响应的舵机
python3 setup_servos.py
# 选择 1，只配置未响应的舵机 ID
```

### 问题 4: ROS 话题不存在

```bash
# rostopic list 看不到 /servo_angles

# 检查 1: roscore 是否运行
ps aux | grep roscore

# 检查 2: servo_reader.py 是否运行
ps aux | grep servo_reader

# 检查 3: 重启节点
# 终端 1: Ctrl+C 停止 roscore，然后重新运行
roscore

# 终端 2: Ctrl+C 停止 servo_reader.py，然后重新运行
rosrun uarm servo_reader.py
```

### 问题 5: 仿真机械臂不动

```bash
# 移动 U-Arm 但仿真机械臂不动

# 检查 1: 话题是否发布
rostopic hz /servo_angles
# 应该看到频率约 50 Hz

# 检查 2: 数值是否变化
rostopic echo /servo_angles
# 移动 U-Arm，观察数值是否变化

# 检查 3: 重启仿真
# Ctrl+C 停止 teleop_sim.py
# 重新运行
python3 teleop_sim.py --robot panda
```
