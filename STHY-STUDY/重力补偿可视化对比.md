## 💪 力的流动对比

### 真实从动机械臂（需要补偿）

```
目标位置
   ↓
┌──────────────┐
│ 控制器算法    │
└──────┬───────┘
       │
       ├─→ PD控制力矩: τ_pd = Kp*e - Kd*v
       │
       ├─→ 重力补偿力矩: τ_comp = -m*g*r
       │
       ↓
   总力矩 = τ_pd + τ_comp
       ↓
```

## 📊 数值对比

### 场景: 手臂水平伸直（90度）

#### 真实机械臂（6kg 手臂）

```
重力矩计算:
τ_gravity = m * g * r * sin(θ)
          = 6 * 9.81 * 0.3 * sin(90°)
          = 17.66 N·m

需要的补偿力矩:
τ_compensation = -17.66 N·m

总控制力矩:
τ_total = τ_pd + τ_compensation
        = 20 + (-17.66)
        = 2.34 N·m (实际用于位置控制)

如果不补偿:
手臂会下垂约 17.66 / Kp 弧度
如果 Kp = 100, 下垂 0.177 rad = 10°
```

#### 仿真机械臂（禁用重力）

```
重力矩:
τ_gravity = 0 (已禁用)

需要的补偿力矩:
τ_compensation = 0

总控制力矩:
τ_total = τ_pd
        = 20 N·m (全部用于位置控制)

优势:
控制更简单，响应更快
```

#### 仿真机械臂（保留重力，大刚度）

```
重力矩:
τ_gravity = 17.66 N·m

PD控制力矩:
τ_pd = Kp * error
     = 1000 * 0.1
     = 100 N·m

总力矩:
τ_total = 100 + 17.66 = 117.66 N·m

分析:
PD力矩 >> 重力矩
100 >> 17.66
隐式补偿了重力
```

#### U-Arm（人手）

```
人手提供的力:
F_hand ≈ 重力 + 控制力
       ≈ 6 * 9.81 + 额外力
       ≈ 60 N + 额外力

舵机:
只读取角度，不施加力
τ_servo = 0

人的感知:
自然感觉到重力
自动调整手的力度
不需要计算
```

## 🧪 实验验证

### 实验 1: 仿真中测试重力影响

```python
import gymnasium as gym
import mani_skill.envs
import numpy as np
import time

# 创建环境（保留重力）
env = gym.make(
    "Empty-v1",
    robot_uids="panda",
    render_mode="human",
    control_mode="pd_joint_pos",
)

obs, _ = env.reset()

# 测试1: 手臂水平，不施加控制
print("测试1: 无控制，观察重力影响")
for _ in range(100):
    env.step(np.zeros(8))  # 零力矩
    env.render()
    time.sleep(0.02)
# 结果: 手臂会缓慢下垂（如果没有禁用重力）

# 测试2: 手臂水平，施加PD控制
print("测试2: PD控制，对抗重力")
target = np.array([0, 0, 0, -1.57, 0, 0, 0, 0])
for _ in range(100):
    env.step(target)
    env.render()
    time.sleep(0.02)
# 结果: 手臂保持水平（PD控制器补偿了重力）
```

## 🎓 总结

### 三句话总结

1. **仿真机器人**: 可以禁用重力或用大刚度隐式补偿，不需要显式算法
2. **真实机器人**: 必须显式计算重力补偿，否则手臂会下垂
3. **U-Arm**: 人手自然补偿重力，舵机只读角度，不需要算法

### 核心原理

```
重力补偿的本质:
提供一个与重力大小相等、方向相反的力

三种实现方式:
1. 软件: 禁用重力（仿真）
2. 算法: 计算补偿力矩（真实机器人）
3. 人力: 人手提供力（U-Arm）

U-Arm 的创新:
用最简单的方式（人手）解决了最复杂的问题（重力补偿）
```
